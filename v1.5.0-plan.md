## What's Pending (Command Restructure)

### Goal

Simplify the command surface from 9 commands down to 6. The current commands are confusing — `plan`, `build`, `version add`, `version build` feels like too many steps. The restructure groups things more logically.

### Before → After

| Before | After | Change |
|--------|-------|--------|
| `:cody help` | `:cody help` | No change |
| `:cody plan` | `:cody plan` | No change |
| `:cody build` | `:cody build backlog` | Renamed — clearer what it does |
| `:cody version add` | `:cody build version` | Merged into one command |
| `:cody version build` | `:cody build version` | Merged — router asks "existing or new?" |
| `:cody patch` | `:cody patch` | No change (already done) |
| `:cody refresh` | `:cody refresh` | Enhanced — adds "update docs?" gate at end |
| `:cody refresh update` | _(folded into `:cody refresh`)_ | No longer a separate command |
| `:cody relearn` | _(removed)_ | Redundant with `/cody` slash command activation |

### Final 6-Command Set

| Command | Description |
|---------|-------------|
| `:cody help` | Shows help and all available commands |
| `:cody plan` | Starts the PLAN phase and creates a new Cody Product Builder project |
| `:cody build backlog` | Generates the feature backlog from the plan |
| `:cody build version` | Work on a version — choose an existing version or add a new one |
| `:cody patch` | Lightweight fix or small enhancement, skips full version build cycle |
| `:cody refresh` | Refreshes agent memory about the project. Auto-detects brownfield. Optionally updates docs |

## File-by-File Changes

### 1. CREATE: `.cody/commands/build-backlog.md`

- Content based on current `build.md`
- Frontmatter command renamed to `:cody build backlog`
- Same logic: copies feature-backlog template, generates backlog from plan
- Update the "done" message to reference new commands (`:cody build version` instead of `:cody version build`)

### 2. CREATE: `.cody/commands/build-version.md`

- New router command for `:cody build version`
- Checks if `feature-backlog.md` exists (if not, tells user to run `:cody build backlog` first)
- Asks the USER: "Do you want to work on an existing version or add a new one?"
- If existing → delegates to `build-version-existing.md` (internal)
- If new → delegates to `build-version-new.md` (internal)

### 3. RENAME + MODIFY: `.cody/commands/version-build.md` → `.cody/commands/build-version-existing.md`

- Renamed so all build-related files group together alphabetically
- Update frontmatter: command becomes internal (no longer user-facing)
- Update closing message: replace `:cody refresh update` → `:cody refresh`, `:cody version build` → `:cody build version`, `:cody version add` → `:cody build version`

### 4. RENAME + MODIFY: `.cody/commands/version-add.md` → `.cody/commands/build-version-new.md`
